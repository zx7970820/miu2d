## 1. 访问游戏

## 2. 检查资源加载

### 浏览器开发者工具 (F12)

#### Console 标签
查看是否有 JavaScript 错误：
- 资源加载失败 (404)
- 解析错误
- 运行时错误

#### Network 标签
查看资源请求：
- 筛选 `resources` 查看游戏资源请求
- 检查是否有 404 或 500 错误
- 查看响应时间

## 3. 常见问题

### 问题 1: 资源 404 错误
**症状**: Network 中看到 `/game/enozheng123-gmail-com/resources/xxx` 返回 404

**解决方案**:
```bash
# 检查文件是否在数据库中
wsl bash -c "cat check-resource.sql | docker exec -i miu2d-postgres psql -U postgres -d miu2d_db"

# check-resource.sql 内容：
SELECT name, storage_key, parent_id
FROM files
WHERE game_id = '96670aaf-94d9-40d3-a320-e57285843008'
  AND name = '文件名.msf';
```

### 问题 2: 地图加载失败
**症状**: 游戏启动后黑屏或提示地图不存在

**解决方案**:
```bash
# 检查地图文件
wsl bash -c "docker exec miu2d-postgres psql -U postgres -d miu2d_db -c \"SELECT name FROM files WHERE game_id = '96670aaf-94d9-40d3-a320-e57285843008' AND type = 'file' AND name LIKE 'map_%.mmf' ORDER BY name;\""

# 更新游戏配置中的 initialMap
# 在 Dashboard 的游戏设置中修改
```

### 问题 3: 玩家角色加载失败
**症状**: 提示找不到玩家数据

**解决方案**:
```bash
# 检查玩家配置
wsl bash -c "docker exec miu2d-postgres psql -U postgres -d miu2d_db -c \"SELECT id, name, key FROM players WHERE game_id = '96670aaf-94d9-40d3-a320-e57285843008';\""

# 确保 game_configs.data.playerKey 与 players.key 匹配
```

### 问题 4: 后端服务未响应
**症状**: 所有请求超时或连接失败

**解决方案**:
```bash
# 检查后端服务状态
wsl bash -c "curl -I http://172.23.87.31:4100/health"

# 查看后端日志
cd packages/server
pnpm dev
```

### 问题 5: 前端代理错误
**症状**: CORS 错误或代理失败

**解决方案**:
```bash
# 检查 Vite 配置
cat packages/web/vite.config.ts

# 检查环境变量
cat packages/web/.env.local

# 重启前端服务
cd packages/web
pnpm dev
```

## 7. 性能优化

如果游戏运行缓慢：

1. **检查资源缓存**
   - 响应头应包含 `ETag` 和 `Cache-Control`
   - 浏览器应该缓存已加载的资源

2. **检查数据库查询性能**
   ```sql
   EXPLAIN ANALYZE
   SELECT * FROM files 
   WHERE game_id = '96670aaf-94d9-40d3-a320-e57285843008' 
     AND name = 'mouse.msf';
   ```

3. **检查 MinIO 连接**
   - 确保 MinIO 服务正常运行
   - 检查网络延迟
