# 黑屏问题修复说明

## 问题分析

根据日志分析，游戏加载流程如下：

```
[Loader] Starting new game...
[ScriptExecutor] Executing: LoadMap(map_002_凌绝峰峰顶.mmf)
[ScriptExecutor] Executing: SetPos(15, 60)
[ScriptExecutor] Executing: FadeOut(1000)  ← 这里执行了淡出效果
[Loader] New game started
```

问题原因：
1. 游戏成功加载（地图、玩家、资源都正常）
2. 但是执行了 `FadeOut(1000)` 命令，使屏幕变黑
3. 没有对应的 `FadeIn()` 命令来清除黑屏
4. 导致游戏虽然运行正常，但屏幕一直是黑色的

## 修复方案

更新 `newGameScript` 配置，在 `LoadGame(0)` 后添加 `FadeIn()` 命令：

```sql
UPDATE game_configs
SET data = jsonb_set(
  data,
  '{newGameScript}',
  '"LoadGame(0)\nFadeIn()"'::jsonb
)
WHERE game_id = '96670aaf-94d9-40d3-a320-e57285843008';
```

## 修复结果

- 修复前：`newGameScript = "LoadGame(0)"`
- 修复后：`newGameScript = "LoadGame(0)\nFadeIn()"`

现在游戏启动流程：
1. 执行 `LoadGame(0)` 加载初始存档
2. 执行 `FadeIn()` 淡入效果，清除黑屏
3. 游戏正常显示

## 测试步骤

1. 刷新浏览器页面（Ctrl+F5 强制刷新）
2. 点击"开始新游戏"
3. 等待加载完成
4. 应该能看到游戏画面正常显示（不再是黑屏）

## 技术细节

### ScreenEffects 淡入淡出机制

- `fadeTransparency`: 0 = 完全透明（正常），1 = 完全不透明（黑屏）
- `FadeOut()`: 启动淡出效果，`fadeTransparency` 从 0 增加到 1
- `FadeIn()`: 启动淡入效果，`fadeTransparency` 从 1 减少到 0
- 淡入淡出速度：1.8/秒，完整淡入/淡出约需 550ms

### 为什么之前的修复无效

之前尝试直接修改存档数据设置 `screenFade: 0`，但这只影响存档加载后的状态。
问题在于 `newGameScript` 执行流程中的 `FadeOut` 命令，需要在脚本层面添加 `FadeIn` 来解决。
